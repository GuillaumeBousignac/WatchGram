# MVP Definition – Phase 2

## Phase 2 Objective

The objective of Phase 2 is to introduce social and transactional features to the application,
allowing users to communicate with each other, share content, and buy or sell watches directly
through the platform.

---

## Main User Flow

1. The user logs into their account
2. The user adds contacts who are already using the application
3. The user views shared publications from their contacts
4. The user starts a real-time conversation with another user
5. The user accesses the watch marketplace
6. The user lists a watch for sale or selects a watch to buy
7. The payment is completed within the application

---

## Essential Screens

- Login / Sign-up screen
- Contacts list
- Private messaging screen
- Publications feed
- Publication detail screen
- Marketplace screen
- Watch detail screen
- Payment screen

---

## Minimal Features

- Add and manage contacts
- One-to-one real-time messaging
- Create and view publications
- List a watch for sale (title, description, price, images)
- Browse watch listings
- Purchase a watch
- Secure in-app payment

---

## Explicitly Excluded

- Group chat
- Audio or video calls
- Ratings and review system
- Negotiation system (offers / counter-offers)
- Installment payments
- Multi-currency support
- Integrated shipping management
- Advanced push notification system
