2.  Prototype Overview

- This prototype aims to simulate the complete user experience of WatchGram before the development phase.
- It allows testing of the main user flows:

Discovering watches through the feed
Logging in with acceptance of the terms and conditions
Browsing profiles and posts
Creating a listing using the “+” button
The prototype is interactive and navigable like a real application.
Screens Included in the Prototype

1. Login Screen
Simulated features:
Email and password input
Link to the terms and conditions
Checkbox: “I accept the terms and conditions”
Interactions:
If the user accepts → access to the feed
If the user refuses → automatic return to the login screen
➡️ Simulates a realistic legal constraint.

2. Main Feed

Simulated features:
List of watch posts
Each post displays:
Image
Brand and model
Price
Case size
Movement type
Interactions

Buttons:
❤️ Like
💬 Comment
⭐ Add to favorites
🔁 Add to comparison

Click actions:
Image → watch detail page
Brand → brand profile page
➡️ Simulates content discovery similar to Instagram.

3. Watch Detail Page

Features:
Image gallery
Full specifications
Detailed description
Interactions:
Back to feed
Add to favorites
Compare with other watches

4. Post Creation ( “+” Button )

Simulated features:
Image upload (drag & drop)
Preview
Complete form:
Name
Brand
Price
Diameter
Movement
etc.
Interactions:
“Publish” button → success pop-up
“Reset” button → clears the form
Individual image removal

➡️ Represents the core of content creation.

Main Supported Interactions
The prototype allows testing of:
Navigation between screens
Complete user flow:
Login → Feed → Detail → Post creation
Realistic behaviors:
Terms rejection
Form errors
Visual feedback
Value of the Prototype Before Development
This prototype allows:
Validation of usability and ergonomics
Verification of user flow logic
Anticipation of UX issues
Presentation of the project without writing code

➡️ It serves as a functional reference for the final development of the WatchGram website.
