Screen 1 – Main Feed (WatchGram)
Screen Description

The main feed is the central screen of the WatchGram app.

It displays a vertical list of posts featuring watches, presented as cards centered on the screen.

Each post contains:

- Main image of the watch (with navigation if there are multiple images)
- Brand (e.g., Rolex) and account status (official)
- Model name
- Key information: price, diameter, movement type
- Number of likes
- Publication date

Social interactions:
- Like (heart icon)
- Comment
- Share
- Add to favorites (bookmark icon)
- Field to add a comment
Main button: "+ Add to comparison"

Quick access:
- Access to the brand profile via the avatar
- Direct access to comments
- Access to the comparison tool

Overall navigation:

A fixed navigation bar at the top provides access to:
- Power supply
- Favorites (with counter)
- Comparison (with counter)
- Messages (with notification badge)
- User account
- A floating button at the bottom right allows you to create a new post.

UX/UI Design Justification
Social Media Inspiration

The feed adopts Instagram's conventions:
- infinite vertical scrolling,
- central card with shadow and rounded edges,
- actions visible below each post.

Objective: to make the interface immediately familiar, requiring no learning curve.

Image as the Main Element
The photo occupies the majority of the car card:
- the watch is a visual and emotional product,
- the user's initial decision is based on visuals,
- technical information is then provided as a supplement.
- The large image format highlights:
- the materials,
- the finishes,
- the dial design.

Information Prioritization
Intentional Order:
- Image
- Brand/Model
- Likes and Interactions
- Information Techniques
- Comparison Button
This respects the natural user journey:
discovery → interest → decision → action

Comparison Integrated into the Feed
The "Add to Comparison" button is present directly on each post to:
- avoid loss of context,
- allow for progressive selection,
- transform the feed into a decision-making tool.

Comparison thus becomes a natural feature, not an isolated page.

Persistent Navigation
The top bar is:
- sticky (always visible),
- accessible on desktop and mobile,
- consistent across all pages.

The counters (favorites, comparison, messages) provide immediate feedback to the user.

Objective: never trap the user on a page with no way out.

Overall screen objective

The main feed must:
- maximize visual discovery,
- encourage social interaction,

prepare for comparison and purchase decision,

while maintaining a simple, fluid, and intuitive interface.






--/Screen 2 – Login Page/--
Screen Description:

The login page allows the user to access their Watchgram account.

It contains:
- Application logo and name
- Username field
- Password field
- "Forgot password" link
- Main "Login" button

Link to the registration page

The interface is centered on the screen in a card format with a glassmorphic effect.

Justification of UX/UI Choices
Premium Design

The dark style with lighting effects and metallic gradients evokes:

- luxury,
- fine watchmaking,
- premium quality.

This reinforces the Watchgram brand image.

Focus on Action
The page contains only one main action:
- Login

Objective:
- Reduce cognitive load,
- Avoid distractions,
- Naturally guide the user. - Clear visual hierarchy

Intentional order:
- Logo / name
- Input fields
- Login button
- Secondary links
This corresponds to the natural path of the eye.

Immediate feedback
In case of error:
- Visible message,
- Automatic disappearance,
- Button disabled during the request.

→ The user always understands what is happening.

Accessibility
Form fields

Each input is associated with a <label>:
- Screen reader compatible,
- Improves mobile UX.

Contrast
- Light text on a dark background.
- Highly visible button.
→ WCAG AA compliant.

Keyboard navigation
- Logical tabbing.
- Button accessible without a mouse.

Readability
- Sans-serif font.
- Comfortable size.
- Generous spacing.

Error messages
- Clear and explicit text. - Color + message (not just color).
→ Accessible for colorblind users.

Screen objective:
To allow the user to:
- log in quickly,
- feel confident,
- perceive Watchgram as a serious and premium platform.

Concluding sentence:
The login page aims to establish a climate of trust and quality from the very first contact with the application, while guaranteeing a simple, accessible, and efficient experience.

Screen 3 – Creating a Post (+ Button)
Screen Description

This screen allows users to post a watch ad on WatchGram.

It is accessible via the “+” button in the navigation bar, similar to Instagram.

Users can:
Add up to 5 photos of the watch
-Enter the main information:
-Name
-Brand
-Price
-Diameter
-Movement
-Add optional features:

-Thickness
-Water resistance
-Material
-Free description
An image preview is displayed before posting.

Justification for Design Choices
1. Organization into Sections
The form is divided into 3 parts:
-Watch Photos
-Basic Information
-Price and Features
➡️ This avoids overwhelming the user and makes data entry clearer.

2. Drag & Drop for Images
Users can drag and drop their photos or click.
Just like on social media, it's more intuitive and faster.

3. Preview with Delete
Each image can be deleted individually before publishing.
The first image is marked as "Main."

➡️ Users retain control over what is displayed in their feed.

4. Visual Feedback
- Red borders for errors
- Messages under each invalid field
- Success pop-up after publishing
➡️ Users immediately understand what's happening.

5. Clear Buttons
- Reset: Clear everything
- Publish: Send the post
The buttons are large and clearly visible, optimized for mobile devices.

Accessibility Decisions

1. High Contrast
Dark background + light text → readable even in low light.

2. Clear Labels
Each field has a clear label:
“Watch Name”, “Brand”, “Price (€)”, etc.
➡️ Screen reader compatible.

3. Text-Based Error Messages
Errors are not only color-coded:
they are also displayed in text.
➡️ Accessible for colorblind users.

4. Required Fields Identified
Essential fields are marked with an asterisk (*).
➡️ The user knows what to fill in first.

5. Mouse-Free Operation
Everything can be done with the keyboard:
- Tab navigation
- Form submission
- Image deletion

Role of this screen in the application

This screen is the heart of WatchGram's content creation.

Without it, there is no feed.

It allows:
- to populate the platform
- to standardize ads
- to have comparable data (price, size, movement, etc.)

➡️ It's the equivalent of the "Publish" button on Instagram, but adapted for watches.
